Infinity Nakshatra — deploy Google Apps Script (includes upsertPlotRow)

The API key is already set in:
  - apps_script_collection_data.gs  →  SCRIPT_API_KEY
  - infinity_nakshatra_dashboard.html → COLLECTION_API_KEY
They must stay identical.

You still must paste the Web app URL into the HTML (COLLECTION_API_URL) after step 6.

Steps
-----
1. Open your Google Sheet → Extensions → Apps Script.
2. Replace the project code with the full contents of: apps_script_collection_data.gs
   (SCRIPT_API_KEY is already filled — or change it in BOTH files if you want a new secret.)
3. Save the Apps Script project (disk icon).
4. Deploy → New deployment → Select type: Web app.
   - Execute as: Me
   - Who has access: Anyone  (simplest for the portal; tighten later if needed)
   In appsscript.json use "access": "ANYONE_ANONYMOUS" (required for Cloudflare /api/collection relay).
5. Deploy. Copy the Web app URL (must look like …/macros/s/…/exec).
6. In infinity_nakshatra_dashboard.html set:
     const COLLECTION_API_URL = "PASTE_THE_FULL_URL_HERE";
   Save the file.
7. Serve the portal over http://127.0.0.1:8080/… (not file://) and test:
   - Owner contact update
   - Admin → Unsold → Update → Save to Google Sheet
   - Backup payments / collection rows

If you rotate the key, update BOTH SCRIPT_API_KEY and COLLECTION_API_KEY on the same value.

----------------------------------------------------------------------
"Failed to fetch" even with correct /exec URL (browser cannot reach Google)
----------------------------------------------------------------------
Some networks block or break HTTPS to script.google.com from the browser only.
Use the small local relay (Python forwards the POST; often works on the same PC).

1. In PowerShell (from the Nakshatra folder), set your real Web App URL and start the relay:
     $env:APPS_SCRIPT_POST_URL="https://script.google.com/macros/s/…/exec"
     python scripts/collection_proxy.py
   Leave that window running. You should see: listening http://127.0.0.1:8765

2. In infinity_nakshatra_dashboard.html set:
     const COLLECTION_API_URL = "http://127.0.0.1:8765";

3. Open the portal over http://127.0.0.1:8080/… (not file://, not https), same machine.
   If the page is https, the browser blocks http://127.0.0.1 (mixed content).

4. Test backup or sheet update again.

Cloudflare Pages (https://nakshatra-portal.pages.dev)
----------------------------------------------------------------------
Production cannot POST directly to script.google.com from the browser.
The repo includes a same-origin relay:

  Portal  →  POST /api/collection  →  Google Apps Script /exec

Files: functions/api/collection.js + nakshatra_collection_exec_url.js
(auto-uses /api/collection on *.pages.dev hosts).

Push to GitHub → Cloudflare Pages redeploys → then:
  1. Open https://nakshatra-portal.pages.dev/api/collection
     (should say "Nakshatra collection proxy OK").
  2. Hard-refresh the dashboard (Ctrl+F5).
  3. Admin FY filter should show "N payment row(s) from Sheet".

Optional: Cloudflare Pages → Settings → Environment variables:
  APPS_SCRIPT_EXEC_URL = your full /exec URL

IMPORTANT — Web app access for Cloudflare relay:
  Apps Script → Deploy → Manage deployments → Edit (pencil)
  - Execute as: Me
  - Who has access: Anyone   (NOT "Anyone with Google account")
  In appsscript.json use "access": "ANYONE_ANONYMOUS".

IMPORTANT — Owner login OTP email (required once):
  1. Open the Google Sheet → Extensions → Apps Script.
  2. In the function dropdown choose: authorizeInfinityNakshatraMail
  3. Click Run ▶ → Review permissions → Allow (Gmail + Mail + Sheets).
  4. Check the society Gmail inbox for a test message "login OTP" with code 123456.
  Use the SAME Google account that owns the sheet and deployed the Web app.
  Corporate/work emails (e.g. company Google Workspace) often BLOCK script email — use society Gmail.

  After authorization, Admin → Profile → Test OTP email to verify from the portal.

If Apps Script shows "Unable to open the file":
  The script project is owned by a different Google account (often the developer).
  Either sign in as that account, OR ask them to Share the script as Editor,
  OR use the Brevo option below (no Apps Script editor needed).

OPTION B — Brevo email (recommended if Apps Script mail cannot be authorized):
  1. Free account: https://www.brevo.com (300 emails/day free)
  2. Create an API key (SMTP & API → API keys)
  3. Verify sender email in Brevo (must match society email)
  4. In the Google Sheet, tab portal_config (created on first portal load), add:
       BREVO_API_KEY        | xkeysib-your-key-here
       BREVO_SENDER_EMAIL   | your-verified-society@gmail.com
  5. Admin → Profile → Test OTP email
